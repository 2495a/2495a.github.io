git

作者:Hu Junming    发布于: 2019-02-21

本篇文章根据自己本身实际对git的操作以及经验做出简单的归纳总结

简述git流程及操作

1. 创建本地仓库

(1) 创建Warehouse文件夹,cd进文件夹输入 git init 初始化仓库

(2) 在Warehouse文件夹中创建a.txt并且写入内容

(3) 终端输入git add a.txt 命令将工作区中修改的内容提交到暂存区

(4) 继上一步后输入git commit -m “附加说明” 命令将暂存区中所有内容提交到仓库(当前分支)

(5) 我们可以在所有阶段使用git status查看仓库当前状态

(6) 当我们修改过a.txt却忘了修改的什么内容 可以用git diff命令查看

(7) 提交过很多次内容后可以使用 git log命令看到我们的提交日志

(8) 当我们想回到上一个版本 git reset –hard head^ head^^(回退两个版本) head~100(回退指定版本)

(10) 使用git checkout – a.txt 可以撤回在工作区的最后一次操作

(11) 如果我们想要删掉某个文件可以使用 git rm –a.txt

(12) 如果错删了或者改错了只要版本库中还存在就可以用 git checkout – a.txt一键还原

以下是在廖雪峰老师网站看到的一张图 简单易懂

2.远程仓库

这里我们介绍使用SSH协议的远程仓库

(1)创建SSH Key

终端输入 ssh-keygen -t rsa -C “youremail”生成密钥 C:\Users\Administrator.ssh 在此路径下
$ ssh-keygen -t rsa -C "youremail"

(2)New SSH Key

登录github：个人头像 -> Settings -> SSH and GPG keys -> New SSH Key 将公钥id_rsa.pub复制粘贴

(3)在github创建仓库

登录github：个人头像 -> Your repositories -> New

(4)关联本地仓库与远程仓库

$ git remote add origin 后面加上你的SSH地址 (如：git remote add origin git@github.com:2495a/gitstudy.git)

(5)将本地库的内容推送的远程库

$ git push -u origin master

(6)之后提交可以使用 git push origin master

(7)从远程库拉取内容到本地库

$ git clone 后面加上你的SSH地址(如：git clone git@github.com:2495a/gitstudy.git)

3.分支

(1)创建分支

$ git checkout -b dev 创建名为dev的分支并切换到dev分支

(2)查看分支

$ git branch 列出所有分支

(3)切换分支

$ git checkout master 切换到master分支

(4)合并分支

$ git merge dev 现在处于master分支 与dev分支合并

(5)删除分支

$ git branch -d dev 删除dev分支

4. 多人协作

(1)将合作者的公钥添加到个人的账户中

(2)在github中添加成员

登录github：找到你的仓库 -> Settings -> Collaborators -> Add Collaborator

(3)合作者拉取远程仓库中的项目

(4)项目成员在本地创建dev分支 在dev分支的基础上分别创建更多分支进行项目的开发

(5)合并分支(可以时不时的进行分支合并)

以上也有一些命令没有说到 请遇到相应的情况自行搜索
